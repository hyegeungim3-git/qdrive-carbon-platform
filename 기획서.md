## 1. 문제 정의

시내버스는 도시 수송부문 탄소 배출의 큰 축이지만, 지자체·운수사의 감축 수단은 사실상 "차량 교체(전기버스)" 하나뿐이다. 교체는 대당 수억 원이 들고 보조금 물량에 묶인다. 반면 **운전 행동 개선(에코드라이빙)은 실증 문헌 기준 연비 3~7%를 즉시, 추가 하드웨어 없이** 만들 수 있는데도 다음 세 가지가 없어 방치된다:

1. **측정** — 절감을 숫자로 증명할 신뢰 가능한 데이터 체계
2. **행동 변화** — 기사가 매 순간 실천하게 만드는 현장 도구
3. **회수 구조** — 절감이 운수사 손익과 지자체 실적으로 연결되는 고리

QDrive는 **이미 전 차량에 의무 장착된 OBD·DTG 단말**의 데이터로 이 세 고리를 잇는다. 신규 장비 투자 0원이 도입 장벽을 없애는 핵심 전제다.

## 2. 타겟과 가치 제안

| 이해관계자 | 페인 포인트 | QDrive가 주는 것 | 화면 |
|---|---|---|---|
| **지자체 관제** | 감축 실적 증빙·관리 수단 부재 | 실측 기반 통합 관제 + 자동 보고 | 관제 대시보드 (8페이지) |
| **버스 기사** | 잔소리식 안전교육, 동기 부재 | 내비형 실시간 코칭 + 랭킹·배지·인센티브 | 기사앱 (8″ 태블릿) |
| **운수사 경영진** | 유가·정비비 압박, 전환 투자 판단 근거 부재 | 월 손익 효과 정량화 + 전환 시뮬레이션 | 경영 리포트 |
| **시민** | 대중교통의 기여가 보이지 않음 | 성과 공개 + 내 노선 등급 조회 | 시민 대시보드 (모바일) |

**한 줄 가치 제안**: "안전운전이 곧 탄소중립입니다 — 이미 달려 있는 단말로, 측정하고, 바꾸고, 회수합니다."

## 5. 차별화 — 왜 QDrive인가 (경쟁 구도와 해자)

### 경쟁 구도

| 경쟁군 | 그들이 하는 것 | 못 하는 것 |
|---|---|---|
| 기존 FMS/BMS 관제 (버스운행관리시스템) | 위치·배차 관제 | 탄소는 사후 리포트 수준, 행동 변화 루프 없음 |
| DTG 운행기록 분석 서비스 | 위험운전 통계·법정 보고 | 연료(OBD)와 결합한 절감 정량화 없음, 기사 동기 설계 없음 |
| 에코드라이빙 교육 업체 | 일회성 교육 | 효과가 수 주 내 소멸 (상시 코칭·측정 부재) |

### 해자 3종 (모방하기 어려운 것)

1. **데이터 폐루프** — OBD 연료분사량 × DTG 주행기록의 **교차 검증 절감 산정 방법론**과 노선별 12개월 베이스라인. 데이터가 쌓일수록 예측·코칭이 정교해지는 축적 우위. 경쟁사가 화면은 베껴도 베이스라인 이력은 못 베낀다.
2. **감축의 화폐화 파이프라인** — 절감량 → KOC 외부사업 인증 증빙 자동 패키징 → 크레딧 수익. "탄소를 보여주는" 서비스는 많지만 "탄소를 돈으로 바꿔주는" 운영 체계는 없다.
3. **멀티 에이전트 운영** — 관제 인력을 늘리지 않고 412대를 관리하는 구조. 에이전트가 밤새 분석·조치안을 준비하고 사람은 승인만 한다(HITL). 지자체·운수사의 실제 도입 장벽(운영 인건비)을 정면으로 제거.

### 어필 포인트 (한 문장씩)

- "**대당 도입비 0원** — 이미 달려 있는 단말로 시작합니다"
- "**절감 못 하면 과금도 없습니다** — 성과 공유형 요금"
- "**관제 요원을 늘리지 마세요** — 에이전트가 밤새 일하고, 아침에 승인만 하세요"
- "**감축이 수익이 됩니다** — KOC 크레딧까지 한 파이프라인"

## 6. 비즈니스 모델

| 수익원 | 구조 | 데모 근거 수치 |
|---|---|---|
| ① SaaS 구독 | 대당 월 구독 (관제+기사앱+리포트) | 412대 × 1.5만원 = 월 618만원/시 |
| ② 성과 공유 (Shared Savings) | 검증된 연료비 절감액의 20% — **미절감 시 0원** | 세운버스 월 절감 1,065만원 → 213만원 |
| ③ 크레딧 중개 | KOC 인증 대행 + 거래 수수료 10% | 연 393.7t × 8,900원 × 10% ≈ 35만원 (규모 확대 시 성장) |

- 성과 공유형이 **도입 설득의 핵심**: 고객 리스크 0 구조라 조달·심의를 통과하기 쉽다. ①은 바닥 매출, ②가 성장 엔진, ③은 규모의 옵션.
- **시장 규모**: 전국 시내버스 약 35,000대(1차) → DTG 의무 장착 사업용 차량(화물·전세버스·택시) 약 100만 대(확장). 대구 412대는 0.04%의 교두보.

## 7. AI 에이전트 아키텍처 (화면 A의 설계 근거)

| 에이전트 | 담당 | 밤새 하는 일 | 사람에게 넘기는 것 |
|---|---|---|---|
| 코칭 에이전트 | 기사 486명 | 주행 분석, 코칭 발송, 교육 대상 선별 | 교육 배정안 |
| 정비 에이전트 | 차량 412대 | OBD 이상 패턴 스캔, 고장 예측 갱신 | 정비 예약안 |
| 플래닝 에이전트 | 전환·충전 | 유가·전력단가 반영 시나리오 재계산 | 스케줄 변경안 |
| 보고 에이전트 | 실적·증빙 | 집계, KOC 증빙 패키징, 보고서 초안 | 제출 승인 |

**운영 원칙 3가지** (화면에 그대로 반영됨):
1. **Human-in-the-loop** — 에이전트는 조치를 '준비'까지만, 실행은 항상 사람 승인 후
2. **설명 가능성** — 코파일럿은 답만 주지 않고 조회→분석 단계를 공개 (감사 추적 가능)
3. **정직한 시뮬레이션** — 데모임을 화면에 명시, 실서비스 스택(LLM 에이전트 + 정비·충전·보고 API 도구 연결)을 함께 안내

## 8. 핵심 설계 결정 (Why)

1. **죽은 컨트롤 금지** — 클릭 가능한 모든 것은 반응한다 (기간 토글·알림 패널·시뮬레이터는 실동작, 정식 기능 밖 버튼은 데모 토스트). 라이브 데모의 신뢰는 여기서 갈린다.
2. **시뮬레이터는 한계 체감 모델** — 선형(대수×상수)이 아니라 효과 큰 차량부터 선정해 체감하는 곡선. "AI가 대상을 고른다"는 서사를 수학이 뒷받침한다.
3. **이해관계자별 언어 번역** — 같은 절감량이 관제에선 tCO₂, 경영진에겐 원(₩), 시민에겐 소나무 그루로 표현된다. 밀도 모드도 다르다 (관제 고밀도 / 시민 저밀도·모바일).
4. **정직한 시뮬레이션** — 랜딩·시민 푸터에 목업임과 산정 기준을 명시. 실증 문헌 범위(3~7%)를 벗어나는 수치는 쓰지 않는다.
5. **행동 루프 완결** — 측정(관제) → 행동(기사 코칭+게이미피케이션) → 회수(경영 손익+KOC) → 확산(시민 공개). 어느 화면도 고립되지 않는다.

## 9. 6분 데모 시나리오 (랜딩 '권장 시연 동선'과 동일)

1. **통합 운영** — 움직이는 차량·히트맵으로 "지금 어디서 새는가" (반월당 붉은 구간 지목)
2. **기사앱** — 같은 반월당의 감속 코칭. "관제의 데이터가 현장의 행동이 됩니다" + 랭킹·배지
3. **탄소·연료** — 기간 토글 일→월→연. "행동이 누적되면 이렇게 쌓입니다" + 상관 r=0.81
4. **AI 에이전트 센터** — 아침 브리핑 → 정비 예약 승인 클릭 → 코파일럿에 "연비가 가장 많이 떨어진 차량은?" 질문, 단계 공개형 답변. "사람은 승인만 합니다"
5. **AI Planning → 경영 리포트** — 슬라이더 6→24대, 한계효과 곡선. "다음 투자를 데이터로 정합니다" → 월 순효과 +1,175만원
6. **시민 대시보드(폰)** — 소나무 59,600그루, 내 노선 등급. "도시의 자부심으로 돌아옵니다"
